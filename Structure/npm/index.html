What is npm?

expand_more

npm (Node Package Manager) is the world's largest software registry and the default package manager for Node.js. It consists of three main components:

  • The CLI tool for managing packages
  • The online registry of packages
  • The website for discovering packages

Installation & Setup

expand_more

npm comes pre-installed with Node.js. To verify your installation:

  • terminal node --version - Check Node.js version
  • terminal npm --version - Check npm version
  • update npm install -g npm@latest - Update npm globally

Important Files & Folders

expand_more
  • description
    package.json - Project metadata, dependencies, scripts
    Contains project info, dependencies, and custom scripts
  • lock
    package-lock.json - Dependency lock file
    Ensures consistent installations across environments
  • folder
    node_modules/ - Installed packages directory
    Contains all downloaded dependencies (don't commit to Git)
  • block
    .npmignore - Files to ignore when publishing
    Similar to .gitignore but for npm publishing

Best Practices & Tips

expand_more
  • check_circle
    Use package-lock.json: Always commit this file to ensure consistent installations across environments
  • security
    Regular Security Audits: Run npm audit regularly to check for vulnerabilities
  • construction
    Dev Dependencies: Use --save-dev for packages only needed during development
  • speed
    Use npx: Run packages without installing them globally with npx <package>
  • update
    Keep Dependencies Updated: Regularly update packages to get security fixes and new features
  • folder
    Don't Commit node_modules: Add it to your .gitignore file
warning
Global Installation Warning: Avoid installing packages globally unless they're CLI tools you'll use across multiple projects.

Package.json Scripts

expand_more

Common script configurations in package.json:

  • play_arrow "start": "node index.js" - Start the application
  • bug_report "test": "jest" - Run tests
  • build "build": "webpack --mode production" - Build for production
  • developer_mode "dev": "nodemon index.js" - Development server
  • format_paint "format": "prettier --write ." - Format code
  • rule "lint": "eslint ." - Lint code

Troubleshooting

expand_more
  • delete_forever
    Clear npm cache: npm cache clean --force
    Fixes corrupted cache issues
  • refresh
    Reinstall node_modules: Delete folder and run npm install
    Fixes dependency conflicts
  • admin_panel_settings
    Permission issues: Use sudo or fix npm permissions
    Avoid using sudo when possible
  • network_check
    Network issues: Check proxy settings or use different registry
    npm config set registry https://registry.npmjs.org/
